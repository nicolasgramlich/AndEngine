EngineLock org.andengine.engine.Engine.getEngineLock ( )

The EngineLock can be used to EngineLock#lock()/EngineLock#unlock() on, to ensure the code in between runs mutually exclusive to the UpdateThread and the GLThread. When the caller already is on the UpdateThread or the GL-Thread, that code is executed immediately.

Returns
the EngineLock the Engine locks on to ensure mutually exclusivity to the UpdateThread and the GLThread.
void org.andengine.engine.Engine.runOnUpdateThread ( final Runnable  pRunnable,
final boolean  pOnlyWhenEngineRunning 
)

This method is useful when you want to execute code on the UpdateThread, even though the Engine is paused.

Parameters
pRunnablethe Runnable to be run on the UpdateThread.
pOnlyWhenEngineRunningif true, the execution of the Runnable will be delayed until the next time Engine#onUpdateUpdateHandlers(float) is picked up, which is when Engine#isRunning() is true. if false, the execution of the Runnable will happen as soon as possible on the UpdateThread, no matter what Engine#isRunning() is.
